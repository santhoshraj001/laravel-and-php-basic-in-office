<?php

namespace App\Http\Controllers;
use Illuminate\Support\Facades\DB;

use Illuminate\Http\Request;

class newVueController extends Controller
{
    //
    // Get All Users
    public function index()
    {
        $users = DB::table('vue_store')->get();
        return response()->json($users);
    }

    // Insert User
    public function store(Request $request)
    {
        $request->validate([
            'name' => 'required|string|max:255',
            'email' => 'required|email|max:255',
            'phone' => 'required|string|max:20',
            'place' => 'required|string|max:255',
        ]);
        DB::table('vue_store')->insert([
            'name' => $request->name,
            'email' => $request->email,
            'phone' => $request->phone,
            'place' => $request->place,
            'created_at' => now(),
            'updated_at' => now()
        ]);

        return response()->json(['message' => 'User Added']);
    }

    // Update User
    public function update(Request $request, $id)
    {
        DB::table('vue_store')
            ->where('id', $id)
            ->update([
                'name' => $request->name,
                'email' => $request->email,
                'phone' => $request->phone,
                'place' => $request->place,
                'updated_at' => now()
            ]);

        return response()->json(['message' => 'User Updated']);
    }

    // Delete User
    public function destroy($id)
    {
        DB::table('vue_store')->where('id', $id)->delete();

        return response()->json(['message' => 'User Deleted']);
    }
}

public function login(Request $request)
{
    if (!Auth::attempt($request->only('email', 'password'))) {
        return response()->json(['message' => 'Invalid credentials']);
    }

----what is MD5----
MD5 (Message Digest 5) is a hashing algorithm that converts a file's content into a fixed unique string (hash).
=> A hash converts data into a unique fixed-length value, mainly used for verification, duplicate detection, and password storage.
=> md5_file() generates a unique fingerprint(unique string) from the file's content like bitrate, artist,duration and other details from audio
   not from its filename.

Example:
audio.mp3 ‚Üí MD5 Hash ‚Üí 5d41402abc4b2a76b9719d911017c592 (used to detect duplicate files).
---End of MD5----
---- Difference between hash and encryption and decryption ----
=> Hash (single line): Hashing is used to securely verify data (such as passwords or files) by 
   converting it into a fixed unique value that cannot be reversed.

=> Encryption: Converts readable data into unreadable data using a key.
Example:
Password: mypass123
Encrypted: X7@kP#9Lm

=> Decryption: Converts encrypted data back to the original readable data using the key.
Example:
X7@kP#9Lm ‚Üí mypass123

=> Password Authentication Example:
User password: mypass123
Stored hash: 5f4dcc3b5aa765d61d8327deb882cf99
(Definition of this pass authentication example):
    During login, the entered password is hashed again and compared with the stored hash. 
    If both hashes match, authentication succeeds.

=> Difference in one line: Encryption can be decrypted back to the original value, 
   whereas hashing is one-way and cannot be reversed.
---- end of hast and encryption and decryption differences----

---Meaning of ISRC----
What is an ISRC?
ISRC (International Standard Recording Code) is a unique 12-character code used to identify a specific sound recording.
Example:
USRC17607839
Format:
CCXXXYYNNNNN
CC     = Country Code (US)
XXX    = Registrant Code (RC1)
YY     = Year (76)
NNNNN  = Unique Recording Number (07839)
---- end of ISRC-----
